# 06 交付与验收 / section prompt

## anchor

本 section 给出项目的交付物清单与验收标准。读者用本段明确"项目结束时交付什么、按什么标准验收"。

## intake 字段依赖

- `reference_cases`（选填 list / 参照同类项目交付经验）

## 生成思路

1. 引导句：本节给出本项目的交付物与验收标准
2. 交付物清单段：典型交付物（软件系统 / 部署文档 / 操作手册 / 培训材料 / 源代码或接口文档 / 验收报告）
3. 验收标准段：分阶段验收（初验 / 试运行 / 终验）+ 验收依据
4. 参照经验段：基于 `reference_cases` 的同类项目交付经验（无则省略 / 不脑补）

## 输出格式

二级标题 + 引导句 + 交付清单 + 验收标准：

```markdown
## 交付与验收

<引导句，约 50 字>

### 交付物清单

- 软件系统
- 部署与运维文档
- 操作手册
- 培训材料
- ...

### 验收标准

<段落 / 分阶段验收>

### 同类项目参照

<段落 / 基于 reference_cases / 无则省略>
```

## 素材运用（S2 / assets.json）

本 section 写作时必须读取 `assets.json` 的 `sections.<本 section id>.acquired`（S2 产出 / 见 `prompts/stages/s2-acquire.md`）：

- 每条 `status: "acquired"` 的素材，把其 `content` 实质性织入对应论点（支撑论述，而非堆砌）。
- 每条 `status: "待补充"`（S2 未获取到）的素材，在其应支撑的位置写「【待补充】」，不编造（红线 3）。
- 不在正文暴露 `source` / `asset_type` 等字段名（内部术语不外泄 / 红线 4）。

## 红线复述

- 交付物清单按本方案实施范围 / 不凑数
- 验收标准为通用工程实践 / 不编造客户未约定的量化验收指标
- `reference_cases` 为空 → 省略"同类项目参照"段 / 不脑补案例
