# 05 质量与风险保障 / section prompt

## anchor

本 section 给出实施过程的质量保障体系与风险控制措施。读者用本段判断"实施过程可控性"。

## intake 字段依赖

- 无 intake_schema 字段（通用质量 / 风险方法论 / 不依赖客户特定字段）

## CoT 生成思路

1. 引导句：本节给出本项目的质量保障与风险控制措施
2. 质量保障体系段：质量管理方法（如阶段评审 / 测试验证 / 文档规范 / 变更控制）
3. 风险识别与应对段：典型实施风险（数据质量 / 集成对接 / 进度 / 需求变更）+ 应对措施（markdown 表格）
4. 收束句：整体可控性说明

## 输出格式

二级标题 + 引导句 + 质量段 + 风险表：

```markdown
## 质量与风险保障

<引导句，约 50 字>

### 质量保障体系

<段落>

### 风险识别与应对

| 风险类别 | 风险描述 | 应对措施 |
|---|---|---|
| 数据质量 | ... | ... |
| 系统集成 | ... | ... |
| 实施进度 | ... | ... |
| 需求变更 | ... | ... |
```

## 素材运用（S2 / assets.json）

本 section 写作时必须读取 `assets.json` 的 `sections.<本 section id>.acquired`（S2 产出 / 见 `prompts/stages/s2-acquire.md`）：

- 每条 `status: "待补充"`（S2 未获取到）的素材，在其应支撑的位置写「【待补充】」，不编造（红线 3）。

## 红线复述

- 质量 / 风险方法论为通用工程实践 / 不编造客户特定的质量指标
- 不承诺具体的质量量化指标（除非客户字面已要求）
- 风险应对措施务实 / 不空泛
